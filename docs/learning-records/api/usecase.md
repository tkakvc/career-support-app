# ユースケース・業務フロー

---

## UC-01　学習記録を作成する

**アクター:** ログイン済みユーザー  
**目的:** 当日または過去の学習内容を記録する  
**事前条件:** JWT 認証済み

**正常フロー:**
1. 学習日・学習内容・学習時間・タグ（任意）を入力して送信する
2. 学習記録が保存され、作成したレコードが返る

**例外フロー:**
- `content` が空または2000文字超 → 400
- `duration` が 0 以下または 1440 超 → 400
- `date` が未来日 → 400
- 存在しない `tagId` を指定 → 404
- 認証トークン不正 → 401

---

## UC-02　学習記録一覧を取得する

**アクター:** ログイン済みユーザー  
**目的:** 過去の学習記録を一覧で確認する  
**事前条件:** JWT 認証済み

**正常フロー:**
1. 一覧を取得する（絞り込み条件は任意）
2. ログインユーザー自身の学習記録のみ返る

**絞り込みの組み合わせ:**
- タグ名のみ指定 → そのタグが付いた全期間の記録を返す
- `from` / `to` のみ指定 → 期間内の全記録を返す
- タグ＋期間を組み合わせて指定可

**例外フロー:**
- 認証トークン不正 → 401

---

## UC-03　学習記録を詳細取得する

**アクター:** ログイン済みユーザー  
**目的:** 特定の学習記録の詳細を確認する  
**事前条件:** JWT 認証済み

**正常フロー:**
1. 対象の学習記録IDを指定して取得する
2. 該当レコードの詳細が返る

**例外フロー:**
- 他ユーザーのレコードIDを指定 → 403
- 存在しないIDを指定 → 404
- 認証トークン不正 → 401

---

## UC-04　学習記録を更新する

**アクター:** ログイン済みユーザー  
**目的:** 誤った記録内容や学習時間を修正する  
**事前条件:** JWT 認証済み、対象レコードが存在する

**正常フロー:**
1. 対象の学習記録IDと更新内容を送信する
2. 更新後のレコードが返る

**備考:**
- `tagIds` を省略した場合、既存のタグ紐付けをそのまま維持する
- `tagIds` に空配列を指定した場合、全タグの紐付けを解除する

**例外フロー:**
- 他ユーザーのレコードIDを指定 → 403
- 存在しないIDを指定 → 404
- バリデーションエラー → 400
- 認証トークン不正 → 401

---

## UC-05　学習記録を削除する

**アクター:** ログイン済みユーザー  
**目的:** 不要な学習記録を削除する  
**事前条件:** JWT 認証済み、対象レコードが存在する

**正常フロー:**
1. 対象の学習記録IDを指定して削除する
2. 削除完了レスポンスが返る

**例外フロー:**
- 他ユーザーのレコードIDを指定 → 403
- 存在しないIDを指定 → 404
- 認証トークン不正 → 401

---

## UC-06　タグ一覧を取得する

**アクター:** ログイン済みユーザー  
**目的:** 学習記録に付与できるタグの一覧を確認する  
**事前条件:** JWT 認証済み

**正常フロー:**
1. タグ一覧を取得する
2. システム提供の default タグ全件 + ログインユーザーが作成した user タグが返る

**例外フロー:**
- 認証トークン不正 → 401

---

## UC-07　タグを作成する

**アクター:** ログイン済みユーザー  
**目的:** 既存タグにない分類で学習内容を整理するため、独自タグを作成する  
**事前条件:** JWT 認証済み

**正常フロー:**
1. タグ名を入力して作成する
2. type: "user"、createdBy: ログインユーザーIDのタグが作成され返る

**例外フロー:**
- 同じユーザーが同名タグをすでに持っている → 409
- `name` が空または50文字超 → 400
- 認証トークン不正 → 401

---

## UC-08　タグを編集する

**アクター:** ログイン済みユーザー  
**目的:** 作成済みの独自タグの名前を変更する  
**事前条件:** JWT 認証済み、対象タグが自分の作成した user タグである

**正常フロー:**
1. 対象タグIDと新しいタグ名を送信する
2. 更新後のタグが返る

**例外フロー:**
- default タグを指定 → 403
- 他ユーザーが作成したタグを指定 → 403
- 存在しないIDを指定 → 404
- 同名タグがすでに存在する → 409
- 認証トークン不正 → 401

---

## UC-09　タグを削除する

**アクター:** ログイン済みユーザー  
**目的:** 不要になった独自タグを削除する  
**事前条件:** JWT 認証済み、対象タグが自分の作成した user タグである

**正常フロー:**
1. 対象タグIDを指定して削除する
2. 削除完了レスポンスが返る
3. そのタグが紐づいていた学習記録のタグ紐付けは解除される（学習記録自体は削除されない）

**例外フロー:**
- default タグを指定 → 403
- 他ユーザーが作成したタグを指定 → 403
- 存在しないIDを指定 → 404
- 認証トークン不正 → 401
