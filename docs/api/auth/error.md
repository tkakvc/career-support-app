# エラー設計

## エラーレスポンス形式

```json
{
  "error": "エラーメッセージ",
  "status": 400
}
```

---

## エンドポイント別エラーケース

### POST /api/auth/signup

| 条件 | ステータス |
|---|---|
| `email` が未入力または形式不正 | 400 |
| `password` が8文字未満 | 400 |
| `displayName` が未入力 | 400 |
| 同じ `email` がすでに登録済み | 409 |

### POST /api/auth/login

| 条件 | ステータス |
|---|---|
| `email` が未入力 | 400 |
| `password` が未入力 | 400 |
| `email` に該当するユーザーが存在しない | 401 |
| パスワードが一致しない | 401 |

---

## 備考

- ログイン失敗時（メールアドレス不存在・パスワード不一致）はどちらも401を返す。どちらが原因かをレスポンスで区別しない（セキュリティ上、情報を与えないため）
