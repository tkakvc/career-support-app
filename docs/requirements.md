# 要件定義書

**プロジェクト名：** 学習記録管理アプリ  
**バージョン：** 1.0  
**作成日：** 2026-05-23

---

## 1. システム概要

学習記録を一元管理する個人向け Web アプリケーション。  
エンジニアとして継続的に成長するための個人用プラットフォームを実現する。

---

## 2. 背景・課題

技術学習において、以下の情報が複数サービスに分散しやすい問題がある。

- 学習記録

また、過去に何を学んだか・何を目指しているか・次に何をやるべきかを忘れやすく、継続的な学習継続が困難になる。

**検討の経緯：** 当初はキャリアシート・タスク管理・技術棚卸し・情報収集媒体・GitHub連携まで含めた統合管理アプリとして構想していた。しかし、これらを全部浅く実装するよりも「学習記録」という1機能を深く作り込む（タグによる分類、ファイル添付、検索条件の組み合わせ、AIによる学習提案）方が、実務で使われる設計パターンを身につける上で効果が高いと判断し、MVPのスコープを学習記録管理に絞った。キャリアシートやタスク管理は、学習記録機能が固まった後の拡張候補として保留する。

---

## 5. 機能要件

### 5.1 機能一覧

| # | 機能名 | 概要 |
|---|---|---|
| F01 | ログイン | JWT 認証によるログイン・ログアウト |
| F02 | ユーザー管理 | ユーザー情報の登録・編集 |
| F03 | 学習記録管理 | 学習内容の CRUD |
| F09 | AI 学習提案 | OpenAI API を利用した学習提案 |
| F11 | ダッシュボード | 各情報の一覧表示 |
| F14 | ファイル添付 | 学習記録へのファイル添付（画像・PDF・テキスト等）、S3保存・ダウンロード対応 |
| F15 | AI 非同期処理 | SQS + Lambda によるAI分析リクエストの非同期化 |

---

### 5.2 機能詳細

#### F01 ログイン

- メールアドレス＋パスワードによるログイン
- JWT トークン発行・検証
- ログアウト（トークン破棄）

#### F02 ユーザー管理

- ユーザー登録（サインアップ）
- プロフィール情報の閲覧・編集

#### F03 学習記録管理

| 項目 | 説明 | 型 | 必須 |
|---|---|---|---|
| タイトル | 学習タイトル | テキスト | ○ |
| 内容 | 学習詳細 | テキストエリア | ○ |
| カテゴリ | Spring / AWS 等 | 選択 | ○ |
| 学習時間 | 学習した時間（h） | 数値 | ○ |
| 学習日 | 学習実施日 | 日付 | ○ |

操作：一覧表示・新規作成・編集・削除・カテゴリ絞り込み

#### F09 AI 学習提案

ユーザーの学習記録・技術棚卸し情報を元に OpenAI API へ送信し、次に学ぶべき内容を提案する。

**入力例：**
- Spring Boot 学習中
- React 未経験
- AWS 経験少

**出力例：**
- REST API 実装推奨
- React Hooks 推奨

#### F14 ファイル添付

学習記録1件に対して、複数のファイルを添付・管理できる。

| 項目 | 説明 |
|---|---|
| 対応形式 | 画像（PNG / JPG / GIF）、PDF、テキスト、Excel 等 |
| ファイルサイズ上限 | 1ファイルあたり 10MB |
| 添付数上限 | 学習記録1件あたり 10ファイル |
| 保存先 | S3（本番） / ローカルディスク（開発） |
| DB管理対象 | ファイル名・サイズ・MIMEタイプ・保存キー |

操作：一覧取得・アップロード・ダウンロード・削除

#### F11 ダッシュボード

以下を一覧表示する。

- 最近の学習記録
- AI 提案サマリ

## 6. 非機能要件

| 区分 | 要件 |
|---|---|
| セキュリティ | JWT 認証・パスワードハッシュ化・バリデーション・CORS 制御・API 認可制御・環境変数管理 |
| 可用性 | 個人利用前提のため高可用性は対象外（AWS EC2 シングル構成） |
| 性能 | 個人利用前提のため大規模アクセスは対象外 |
| 保守性 | レイヤードアーキテクチャによる関心分離、Docker による環境再現性確保 |
| スケーラビリティ | MVP フェーズでは対象外。AWS 構成で将来的な拡張は可能とする |
| 開発効率 | AI 活用を前提とした開発効率化 |
