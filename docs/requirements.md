# 要件定義書

**プロジェクト名：** 学習・キャリア統合管理アプリ  
**バージョン：** 1.0  
**作成日：** 2026-05-23

---

## 1. システム概要

学習、キャリア、技術情報収集、タスク管理を一元管理する個人向け Web アプリケーション。  
エンジニアとして継続的に成長するための個人用プラットフォームを実現する。

---

## 2. 背景・課題

技術学習やキャリア形成において、以下の情報が複数サービスに分散しやすい問題がある。

- 学習記録
- キャリア整理
- 情報収集
- タスク管理
- 技術棚卸し

また、過去に何を学んだか・何を目指しているか・次に何をやるべきかを忘れやすく、継続的な学習継続が困難になる。

---

## 3. 目的

| # | 目的 |
|---|---|
| 1 | 学習内容を一元管理する |
| 2 | キャリア情報を整理する |
| 3 | 技術情報収集を集約する |
| 4 | AI による学習補助を行う |
| 5 | GitHub 活動を可視化する |
| 6 | モダン Web アプリケーション構成を学習する |

---

## 4. 想定ユーザー

| 区分 | 対象 |
|---|---|
| メイン | Web エンジニア |
| サブ | 転職活動中エンジニア |

### 5.2 機能詳細

#### F01 ログイン

- メールアドレス＋パスワードによるログイン
- JWT トークン発行・検証
- ログアウト（トークン破棄）

#### F02 ユーザー管理

- ユーザー登録（サインアップ）
- プロフィール情報の閲覧・編集

#### F03 学習記録管理

| 項目 | 説明 | 型 | 必須 |
|---|---|---|---|
| タイトル | 学習タイトル | テキスト | ○ |
| 内容 | 学習詳細 | テキストエリア | ○ |
| カテゴリ | Spring / AWS 等 | 選択 | ○ |
| 学習時間 | 学習した時間（h） | 数値 | ○ |
| 学習日 | 学習実施日 | 日付 | ○ |

操作：一覧表示・新規作成・編集・削除・カテゴリ絞り込み

#### F04 タスク管理

| 項目 | 説明 | 型 | 必須 |
|---|---|---|---|
| タイトル | タスク名 | テキスト | ○ |
| 内容 | 詳細説明 | テキストエリア | - |
| 優先度 | 高 / 中 / 低 | 選択 | ○ |
| ステータス | 未着手 / 進行中 / 完了 | 選択 | ○ |
| 期限 | 期限日 | 日付 | - |

操作：一覧表示・新規作成・編集・削除・ステータス更新

#### F05 キャリアシート管理

管理する内容：

- 数年後の目標
- やりたい技術
- 振り返り
- 仕事でやったこと
- 強み / 弱み
- 転職理由整理

操作：閲覧・編集（自由記述中心）

#### F06 技術棚卸し

| 項目 | 説明 | 型 | 必須 |
|---|---|---|---|
| 技術名 | Vue3 / Spring Boot 等 | テキスト | ○ |
| 経験年数 | 実務・学習年数 | テキスト | ○ |
| レベル | 初級 / 中級 / 上級 | 選択 | - |
| 備考 | 補足情報 | テキスト | - |

操作：一覧表示・新規作成・編集・削除

#### F07 情報収集媒体管理

管理対象：Qiita / Zenn / Udemy / paiza / Ping-t / YouTube / Connpass / GitHub 等

| 項目 | 説明 |
|---|---|
| サービス名 | 媒体名 |
| URL | リンク |
| カテゴリ | 動画 / 記事 / 資格 等 |
| メモ | 利用用途メモ |

操作：一覧表示・新規作成・編集・削除

#### F08 GitHub 連携

GitHub API を利用して以下を取得・表示する。

- Repository 一覧
- Commit 履歴
- Contributions
- Issue 一覧

#### F09 AI 学習提案

ユーザーの学習記録・技術棚卸し情報を元に OpenAI API へ送信し、次に学ぶべき内容を提案する。

**入力例：**
- Spring Boot 学習中
- React 未経験
- AWS 経験少

**出力例：**
- REST API 実装推奨
- React Hooks 推奨

#### F10 AI タスク分解

入力したタスクを OpenAI API でサブタスクに分解する。

**入力例：**「JWT ログインを実装したい」

**出力例：**
- JWT 発行 API
- Security 設定
- Login 画面
- Token 保存処理

#### F14 ファイル添付

学習記録1件に対して、複数のファイルを添付・管理できる。

| 項目 | 説明 |
|---|---|
| 対応形式 | 画像（PNG / JPG / GIF）、PDF、テキスト、Excel 等 |
| ファイルサイズ上限 | 1ファイルあたり 10MB |
| 添付数上限 | 学習記録1件あたり 10ファイル |
| 保存先 | S3（本番） / ローカルディスク（開発） |
| DB管理対象 | ファイル名・サイズ・MIMEタイプ・保存キー |

操作：一覧取得・アップロード・ダウンロード・削除

#### F11 ダッシュボード

以下を一覧表示する。

- 最近の学習記録
- 進行中タスク
- GitHub Contributions
- AI 提案サマリ

## 6. 非機能要件

| 区分 | 要件 |
|---|---|
| セキュリティ | JWT 認証・パスワードハッシュ化・バリデーション・CORS 制御・API 認可制御・環境変数管理 |
| 可用性 | 個人利用前提のため高可用性は対象外（AWS EC2 シングル構成） |
| 性能 | 個人利用前提のため大規模アクセスは対象外 |
| 保守性 | レイヤードアーキテクチャによる関心分離、Docker による環境再現性確保 |
| スケーラビリティ | MVP フェーズでは対象外。AWS 構成で将来的な拡張は可能とする |
| 開発効率 | AI 活用を前提とした開発効率化 |

## 7. システム構成（技術スタック）

| 項目 | 技術 |
|---|---|
| Frontend | Vue3 + Nuxt |
| UI Framework | Vuetify |
| Backend | Spring Boot |
| DB | PostgreSQL |
| 認証 | Spring Security + JWT |
| インフラ | Docker / AWS |
| コンテナ実行 | ECS Fargate |
| IaC | Terraform |
| ストレージ | S3 |
| キュー | SQS |
| サーバーレス | Lambda |
| CDN | CloudFront |
| API 連携 | GitHub API / OpenAI API |
| デプロイ | ECS Fargate + RDS + ALB + ACM + CloudFront |

---

## 8. 対象外（やらないこと）

以下はスコープ外とし、MVP では実装しない。

- マイクロサービス
- Kubernetes
- 高度な DDD（Domain-Driven Design）
- Kafka
- 複雑なリアルタイム同期
- 大規模 AI 基盤
