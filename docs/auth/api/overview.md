# 認証API 概要

このAPIは、ユーザーの登録・ログインおよびJWTトークンの発行を行うためのものです。

## 主な機能

- ユーザー登録（サインアップ）
- ログイン・JWTトークン発行

## 想定ユースケース

- 新規ユーザーがメールアドレスとパスワードでアカウントを作成する
- 登録済みユーザーがログインしてJWTトークンを取得し、以降のAPIリクエストに使用する

## 認証方式

| 項目 | 内容 |
|---|---|
| 方式 | JWT（JSON Web Token） |
| アルゴリズム | HS256 |
| トークン有効期限 | 1時間 |
| トークン送信方法 | `Authorization: Bearer {token}` ヘッダー |
| ペイロード | `userId`（UUID）, `iat`（発行日時）, `exp`（有効期限） |

## 共通仕様

| 項目 | 内容 |
|---|---|
| ベース URL | `/api/auth` |
| 認証 | 不要（このAPI自体は認証なしで呼べる） |
| Content-Type | `application/json` |
| レスポンス形式 | JSON |
| 文字コード | UTF-8 |

## セキュリティ方針

- パスワードはBCryptでハッシュ化してDBに保存する。平文は一切保存しない
- JWTの秘密鍵は環境変数で管理し、ソースコードに含めない
- トークンはサーバー側では保持しない（ステートレス認証）
