# ユースケース・業務フロー

---

## UC-01　ユーザー登録をする

**アクター:** 未登録ユーザー  
**目的:** アカウントを作成してアプリを利用できる状態にする  
**事前条件:** なし

**正常フロー:**
1. メールアドレス・パスワード・表示名を入力して送信する
2. アカウントが作成され、JWTトークンが返る
3. 以降のAPIリクエストはこのトークンを使用する

**例外フロー:**
- バリデーションエラー（未入力・形式不正・パスワード短すぎ）→ 400
- 同じメールアドレスがすでに登録済み → 409

---

## UC-02　ログインする

**アクター:** 登録済みユーザー  
**目的:** JWTトークンを取得してAPIを利用できる状態にする  
**事前条件:** アカウントが登録済みであること

**正常フロー:**
1. メールアドレスとパスワードを入力して送信する
2. JWTトークンが返る
3. 以降のAPIリクエストの `Authorization: Bearer {token}` ヘッダーにセットする

**例外フロー:**
- 未入力 → 400
- メールアドレスまたはパスワードが一致しない → 401
